## 1. 结论摘要

- 目标：为跨境卖家（Linkfox AI 视角）提供 Web 版竞品情报监控代理 MVP，把"网页变化"自动解读为「变化 → 战略意图 → 分角色行动建议」的结构化情报，并围绕**「因人而异地推送 + 引用式深度对话」**这一核心价值，支持对单条情报的 AI 深度追问。
- 三大核心亮点（本次重点突出，基本功能不变）：
  1. **信息收集层**：注册后表单收集「我的角色 + 我更关注的信息」，建立**角色标签**（PM/市场/创始人/投资人，可扩展）与**信息标签及权重**（定价/功能/更新日志/招聘/营销活动/合规条款，可自定义偏好），支持后续更改。
  2. **信息匹配层**：由 AI 对网页变化按**六大信息标签**打标，再按用户角色与权重做「人—信息」匹配与个性化排序（有重点、有取舍，但不完全屏蔽其它领域）。
  3. **信息消费层**：在情报详情内以「引用信息 → 对话信息」的方式二次萃取价值，多轮深挖、持续落盘直到用户主动终止对话。
- In / Out 边界：In = 带「角色+信息标签权重」的最简登录/Onboarding + 监控目标管理 + **定时自动采集与解读（UR-3 P0）** + **采集层「markdown 抓取 + JS 注入降级（markdown<3 行触发）」能力（本期默认输入=测试包，对齐原始《生成MD测试用例》）** + 变化检测 + AI 价值定义与信息打标 + 真实大模型分析 + 个性化情报 Inbox（含情报状态：未读/已读/归档，UR-4）+ 引用式多轮深度对话 + **主动通知（邮件推送重点/紧急情报，UR-6 P1）** + 情报反馈（UR-7 P1）；Out = 真实全站规模化爬取与反爬对抗（采集能力已设计，真实抓取可行性见 R-010）、行业政策源采集、竞品自动推荐（UR-2 P2）、Slack/IM 等其它推送渠道（邮件已纳入本期）、多用户权限、质量看板、社媒/流量/AI 提及度信号、跨历史情报追问。
- MVP 边界：3 天可交付；采集层设计为「markdown 抓取 + JS 注入降级」，**本期默认输入用测试包**（`index.html` 基准 + `cases/` 变体），真实站点抓取可行性见 R-010；部署于 Vercel（Serverless 友好托管 DB）。
- 推荐方案：测试包驱动的情报 MVP（Test-Pack-Driven MVP），详见 `requirements/solution.md#2-推荐方案唯一` 与三大亮点 `#21-三大核心亮点本次重点突出`。
- 优先验证点：`R-001`（Vercel 兼容 DB 选型）、`R-002`（3 天工时覆盖度）、`R-003`（大模型密钥/成本）、`R-007`（角色×信息标签匹配与打标可用性）。

## 2. 范围与里程碑

### 2.1 MVP 范围（In / Out）

- **In**：
  1. 最简注册/登录 + 会话保持（单一账号，无邮箱验证/找回密码）。
  2. **〔亮点1·信息收集层〕Onboarding 用户画像**：注册后选择角色标签（PM/市场/创始人/投资人，可扩展自定义角色），系统**自动应用该角色默认信息标签权重**；权重可在设置页按角色分别调整，Inbox 支持角色快切预览不同视角的行动建议。
  3. 监控目标管理：登记竞品（名称 + URL + 赛道：生图/生视频/Agent），支持新增/编辑/删除；采集方式支持「手动即时触发」与「固定时间采集（每日定时）」；系统抓取采用「markdown 提取 + JS 注入降级（markdown<3 行触发）」（本期默认测试包输入，真实抓取见 R-010）；展示监控状态。
  4. **〔UR-3·P0〕定时自动采集与解读**：Vercel Cron 定时对已绑定监控目标自动执行「检测→打标→分析」，无需人工介入；保留手动触发作为补充。
  5. 变化检测：基准 vs 变体文本 diff，过滤纯样式噪音，输出变化候选。
  6. **〔亮点2·信息匹配层〕AI 价值定义与信息打标**：对每条变化候选按六大信息标签（定价/功能/更新日志/招聘/营销活动/合规条款）打标；A/B 测试、纯样式、幻觉诱饵等噪音不生成情报。
  7. AI 分析（真实大模型）：产出变化内容 / 战略意义 / 行动建议（面向当前用户角色个性化 + 通用三视角兜底）/ 优先级三档 / 原文锚点。
  8. **〔亮点2落地〕个性化情报 Inbox（双栏工作区）**：左栏列表 + 右栏 Inspector（详情/深度对话 Tab）；结合角色/权重的个性化排序 + **晨报/核心池/全部**三视图 + 情报状态（未读/已读/归档）+ 筛选（赛道/信息标签/优先级/归档三态）+ Inbox 顶部角色快切。
  9. **〔UR-6·P1〕主动通知（邮件）**：对新产生的紧急/高匹配重点情报邮件推送给对应用户（去重+频控）；支持**多个推送邮箱**；邮件通知在设置中独立配置（可开关、推送时间、推送内容）。
  10. **〔亮点3·信息消费层〕引用式深度对话（Inspector 内嵌）**：在 Inbox 右栏深度对话 Tab 引用一条或多条情报发起多轮对话，支持多会话管理与跨情报联合分析；`/chat` 为兼容重定向路由。
  11. 反馈（UR-7·P1）：情报打标（标签集：有用 / 幻觉 / 漏抓 / 优先级错误 / 建议废话 / A/B 测试误报 / 其他，含问题模块定位+补充说明）并落库；标记「有用」自动加入核心信息池。
- **Out（后续演进）**：真实线上规模化爬取的稳定性/反爬对抗（采集 markdown+JS 注入能力已入本期、默认测试包输入，见 R-010）；行业政策源采集；竞品自动推荐（UR-2 P2）；Slack/IM 等其它推送渠道（邮件已纳入本期）；多用户权限与数据隔离；质量看板与自动回归；社媒/流量/AI 提及度信号；跨历史情报追问；反馈反哺个性化权重；更多角色（技术/运营等）；周报/月报。

### 2.2 里程碑

- **MVP（3 天）**：覆盖全部 P0（登录 → Onboarding 画像 → 监控目标 → 定时自动采集(UR-3) → 变化检测 → AI 打标+分析 → 个性化 Inbox 含归档(UR-4) → 反馈）+ P1（晨报视图、筛选、引用式深度对话、主动通知邮件(UR-6)、反馈改进(UR-7)、改角色/权重）。P2（UR-2 竞品自动推荐）不在本期。
- **M1（后续演进）**：Out 清单中的能力按优先级排期，首选「真实爬取 / 行业政策源 / Slack·IM 推送」之一（见 `solution.md` 后续演进）。

## 3. 核心场景（≤ 3 个）

### 3.1 场景 S-001：设定角色画像、建立监控并首次获得个性化情报〔亮点1+2〕

- **触发**：新用户完成注册后进入 Onboarding，选择角色，随后想开始监控一个竞品。
- **参与者**：登录用户（PM/市场/创始人/投资人或自定义角色）。
- **目标**：完成角色画像（自动配权重），登记竞品并绑定 URL，触发分析后在个性化 Inbox 看到按其画像匹配的结构化情报。
- **成功标准**：
  1. 能在 Onboarding 选择角色，系统自动应用该角色默认信息标签权重。
  2. 能新增/编辑至少 1 个竞品并提供 URL，选择采集方式，系统自动开始监控。
  3. 触发分析后至少生成 1 条含完整字段的情报，且每条被打上 ≥ 1 个信息标签。
  4. 情报出现在个性化 Inbox：与其角色/权重匹配度高的情报被重点置顶，其它领域降权保留而非消失。

### 3.2 场景 S-002：引用式深度对话并反馈〔亮点3〕

- **触发**：用户在 Inbox 打开一条情报，想更深层次地理解它，或联合多条情报分析。
- **参与者**：登录用户。
- **目标**：在 Inspector 查看完整分析与原文来源，引用一条或多条情报多轮追问，并对质量打标反馈（含模块定位）。
- **成功标准**：
  1. Inspector 详情展示完整分析（变化内容/战略意义/通用视角/行动建议·当前角色）+ 可展开「查看原文」。
  2. 可在 Inspector 深度对话 Tab 引用一条或多条情报多轮追问；原文无答案时明确告知"资料不足"。
  3. 支持多会话管理，会话历史持续落盘，终止前不丢失。
  4. 反馈打标含问题模块与补充说明，刷新仍在；标记「有用」自动加入核心池。

### 3.3 场景 S-003：调整画像、快速消费当日重点〔亮点1+2〕

- **触发**：用户当天登录，想快速看当天最重要的情报或核心池中的高价值信号；或调整了关注重心，想让分发随之变化。
- **参与者**：登录用户。
- **目标**：通过晨报/核心池视图与筛选优先看到高优先级情报；在设置或 Inbox 角色快切更改角色/权重后，分发随之调整。
- **成功标准**：
  1. 晨报视图按时间展示当日情报；核心池视图仅展示用户标记的高价值情报。
  2. 可按赛道、信息标签、优先级、归档状态筛选，结果正确。
  3. 在设置中更改角色/权重或 Inbox 快切角色后，Inbox 的个性化排序与行动建议展示随之变化。

## 5. 业务规则与口径（只写影响 AC 的）

- 规则-1（优先级三档）：情报优先级取值为「紧急 / 中等 / 低」三选一。排序默认按此顺序（紧急 > 中等 > 低）。
- 规则-2（去噪）：仅样式/排版（CSS）类变化、A/B 测试导致的文案摇摆、幻觉诱饵（客户数/媒体背书/未上线预告等）不得判定为"有意义变化"，不生成情报或标为无重大变化。
- 规则-3（AI 事实性）：AI 价值定义、打标、分析与对话只能基于「变化候选 + 原文内容」作答，禁止臆造；每条情报须带原文引用锚点（View Source 可跳转）。
- 规则-4（对话兜底）：深度对话的问题在「当前情报 + 被引用原文」中无依据时，必须明确回复"资料不足"，不得编造，不联网现抓。
- 规则-5（持久化）：用户、用户画像（角色+权重）、监控目标、情报（含信息标签）、反馈、对话记录均须持久化到 Serverless 友好 DB；重新部署后不丢失。
- 规则-6（访问控制）：未登录不得访问 Onboarding/Inbox/详情/目标管理/设置等业务页面与接口。
- 规则-7（角色标签口径）：角色为单选，内置取值「产品经理 / 市场营销负责人 / 创业者·创始人 / 投资人」，并支持用户自定义角色；每个角色对应一组默认高关注信息标签（见 `solution.md#亮点-2`）。
- 规则-8（信息标签与权重）：信息标签取值「定价 / 功能 / 更新日志 / 招聘 / 营销活动 / 合规条款」；Onboarding 时按所选角色自动应用默认权重，用户可在设置页按角色分别调整权重；情报须至少命中一个信息标签。
- 规则-9（个性化"有重点、有取舍、不屏蔽"）：Inbox 综合「优先级 + 用户画像匹配度（信息标签权重）」排序，重点领域优先置顶；低权重/非关注领域的情报**降权保留而非隐藏**，避免信息茧房。
- 规则-10（画像变更即时生效）：用户在设置中更改角色或信息权重后，后续 Inbox 分发与排序须即时按新画像计算。
- 规则-11（定时自动采集，UR-3）：系统须由 Vercel Cron **每日定时**自动对已绑定监控目标执行「检测→打标→分析」，无需人工介入即可产出新情报；「手动即时触发」为补充能力，二者产出口径一致（去噪、打标、字段完整性相同）。本期不承诺分钟级「实时」采集。
- 规则-12（情报状态，UR-4）：情报具备状态「未读 / 已读 / 归档」；归档情报默认从主列表隐藏但可通过筛选查看，状态变更须持久化。
- 规则-13（主动通知口径，UR-6）：仅对「紧急」或「与用户画像高匹配」的情报触发邮件推送；同一情报对同一用户只推一次（去重），并做发送频控；用户可在设置「邮件通知」Tab 中配置**多个**推送邮箱、推送时间、推送内容，并可关闭通知。
- 规则-14（核心信息池）：用户可将情报手动加入/移出核心信息池；反馈标记「有用」时自动加入核心池；核心池视图仅展示池中情报，便于聚焦高价值信号。
- 规则-15（Inbox 角色快切）：Inbox 顶部支持切换当前查看角色，即时影响行动建议展示与列表个性化排序，不改变用户持久化画像（除非在设置页保存）。
- 规则-16（反馈标签口径，UR-7，对齐原始《效果评估标准》Bug 反馈规范）：情报反馈标签取值为「有用 / 幻觉 / 漏抓 / 优先级错误 / 建议废话 / A/B 测试误报 / 其他」；除「有用」外均可附问题模块定位与补充说明；「漏抓」用于标记 Visualping 等报变化而本系统未生成情报的场景，「A/B 测试误报」用于标记把 A/B 摇摆误判为真信号的场景。

## 6. 验收标准（AC，可测试）

### 6.1 场景 S-001 的 AC

- AC-001：未注册用户可注册成功；注册后进入 Onboarding，可选择角色（单选）；提交后系统自动应用该角色默认信息标签权重并落库，跳转监控目标页。
- AC-002：登录用户可新增/编辑至少 1 个竞品（名称 + URL + 赛道），并设置采集方式（手动即时触发 / 固定时间采集每日定时）。
- AC-003：点击"立即检测"或定时采集后，系统对竞品 URL 执行变化检测并调用 AI，生成 ≥ 1 条情报。
- AC-004：生成的情报包含全部字段：变化内容、战略意义、行动建议（含面向当前用户角色的个性化建议 + 通用三视角兜底）、优先级（三档之一）、原文引用锚点；且每条被打上 ≥ 1 个信息标签（对应规则-7/8）。
- AC-005：新情报出现在个性化 Inbox；与当前用户角色/权重匹配度高的情报被优先置顶，低权重领域情报仍出现在列表（降权而非隐藏，对应规则-9）。

### 6.2 场景 S-002 的 AC

- AC-006：Inbox Inspector 详情 Tab 展示完整分析（变化内容/战略意义/通用视角/行动建议·当前角色），并提供「查看原文」展开变化对比面板。
- AC-007：在 Inspector 深度对话 Tab 可引用一条或多条情报发起多轮对话；构造原文无依据问题时，回答明确为"资料不足"。
- AC-008：支持多会话管理；对话历史持续落盘，终止前不丢失；刷新后可回看。
- AC-009：对情报打标反馈（标签集：有用/幻觉/漏抓/优先级错误/建议废话/A-B 测试误报/其他，含问题模块定位与补充说明）后，刷新页面反馈状态仍在；标记「有用」后情报进入核心池（对应规则-16）。

### 6.3 场景 S-003 的 AC

- AC-010：晨报视图展示当日情报；核心池视图仅展示核心池情报。
- AC-011：按赛道、信息标签、优先级、归档状态（隐藏/包含/仅看）筛选时，结果集与筛选条件一致。
- AC-012：在设置中更改角色/权重并保存后，或在 Inbox 角色快切后，Inbox 的个性化排序与行动建议展示随之变化。

### 6.4 系统能力 AC（UR-3 / UR-4 / UR-6 补齐）

- AC-013（UR-3 定时自动采集）：配置 Vercel Cron 后，在不点击任何按钮的前提下，定时任务能自动对已绑定监控目标跑通「检测→打标→分析」并产出新情报，字段完整（对应规则-11）。
- AC-014（UR-4 归档）：用户可将情报标记为已读/归档；归档情报默认从主列表隐藏、可经筛选查看；刷新后状态仍在（对应规则-12）。
- AC-015（UR-6 主动通知）：当产生「紧急/高匹配」情报时，系统向设置中配置的推送邮箱发送含所选内容（标题/摘要/行动建议/链接）的邮件；同一情报不重复发送；用户在邮件通知 Tab 关闭后不再收到邮件（对应规则-13）。
- AC-016（部署）：应用部署在 Vercel，浏览器可访问；重新部署后已有数据（用户/画像/情报/状态/对话/反馈）仍存在。

## 7. 异常与边界（只覆盖影响 AC 的关键异常）

- 异常-1（未登录访问）：未登录访问业务页面/接口时，重定向到登录或返回未授权，不泄露数据（对应规则-6）。
- 异常-2（AI 调用失败/超时）：大模型调用失败或超时时，情报状态标记为"分析失败/可重试"，不产生半成品情报，不阻塞其它情报。
- 异常-3（无有意义变化）：基准与变体无实质差异（或仅样式差异）时，明确提示"无重大变化"，不产生噪音情报（对应规则-2）。
- 异常-4（DB 不可用）：数据库不可用时给出明确错误提示，不静默丢数据。
- 异常-5（对话越界）：深度对话超出「当前情报 + 被引用原文」范围时，回复"资料不足"，不联网、不臆造（对应规则-4）。
- 异常-6（画像缺失/未完成 Onboarding）：用户未完成角色/权重设置时，引导其先完成 Onboarding；在画像补全前，Inbox 退化为「仅按优先级排序」的通用视图，不因缺画像而报错或空白。
- 异常-7（无匹配重点情报）：当前画像下无高匹配情报时，仍展示其它领域的降权情报并给出「暂无高相关情报」提示，不呈现空列表（对应规则-9）。
- 异常-8（Cron 定时任务失败/超时，UR-3）：定时采集失败或超时时，记录任务状态为「失败/可重试」，不产生半成品情报，不影响已有情报；下次周期或手动触发可重跑。
- 异常-9（邮件发送失败，UR-6）：邮件发送失败时，情报仍正常入 Inbox（站内可见），通知记为「发送失败」可重试，不因通知失败阻塞情报生成。

## 9. 原型产出判定

- **交互变化结论**：Demo 已验证核心交互——双栏 Inbox 工作区（列表 + Inspector 详情/对话）、Onboarding 角色自动配权重、核心信息池、多情报引用对话、设置两 Tab；可直接进入 design/implementation。
- **页面与入口**：`/login`、`/register`、`/onboarding`、`/targets`、`/inbox`（主工作区，含 `?id=&view=` 参数）、`/inbox/:id` 与 `/chat`（兼容重定向）、`/settings`；侧栏导航仅 Inbox/监控目标/设置（无独立深度对话入口）。
- **关键控件/字段与校验**：
  - Onboarding〔亮点1〕：角色选择（单选枚举，必选）；提交后自动应用默认权重。
  - 竞品登记/编辑：名称（必填）、URL（必填）、赛道（枚举）、采集方式（手动即时触发/固定时间每日定时）。
  - 情报 Inbox〔亮点2〕：晨报/核心池/全部视图；角色快切；Inspector 详情/对话 Tab；核心池标记。
  - 深度对话〔亮点3〕：多情报引用 chip、引用范围、多会话管理、终止对话。
  - 反馈〔UR-7〕：标签（有用/幻觉/漏抓/优先级错误/建议废话/A-B 测试误报/其他） + 问题模块 + 补充说明。
  - 设置〔亮点1+UR-6〕：两个 Tab——角色与权重（含自定义角色、按角色权重弹窗）/ 邮件通知（多邮箱、推送时间、推送内容）。
