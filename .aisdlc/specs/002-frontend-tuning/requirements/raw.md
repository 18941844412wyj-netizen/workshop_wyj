# signal-desk 前端调优

## 背景

signal-desk 是竞争情报监控 Web 应用的前端子项目（React + Vite + TypeScript），位于仓库 `signal-desk/` 目录。当前 MVP 已实现注册登录、收件箱、目标管理、引导流程、设置等核心页面，需要针对前端体验与工程质量进行系统性调优。

## 目标

对 signal-desk 前端进行调优，提升用户体验、界面一致性与可维护性，确保在浏览器端流畅可用、便于后续迭代。

## 调优方向（待澄清细化）

- 页面布局与视觉一致性（Layout、导航、响应式）
- 交互体验（加载状态、空状态、错误提示、表单反馈）
- 组件结构与代码组织（复用、拆分、类型安全）
- 性能与资源（首屏、路由懒加载、静态资源）
- 样式体系（CSS 变量、主题、可访问性基础）

## 约束

- 调优范围限定在 `signal-desk/` 前端，不改变后端 API 契约（除非调优暴露必须修复的对接问题）
- 保持现有功能行为可用，不引入大规模架构重写
- 可与 001-competitor-intel-monitor 规格对齐验收口径

## 交付期望

- 前端调优后的可运行、可构建、可预览版本
- 关键改动有清晰说明，便于评审与回归
